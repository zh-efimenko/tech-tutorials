# Программа обучения: ArchUnit

## Цель

Научиться автоматически проверять архитектурные правила в Spring Boot 3 сервисах с помощью ArchUnit и JUnit 5 — чтобы нарушения выявлялись на этапе сборки, а не на ревью.

## Требования

- Java 17+
- Spring Boot 3.x
- Gradle или Maven
- JUnit 5 (входит в spring-boot-starter-test)
- Базовое понимание слоёв приложения (controller / service / repository)

---

## Уроки

| #  | Тема | Папка | Описание |
|----|------|-------|----------|
| 1  | Что такое ArchUnit | `01_intro/` | Проблема ручного ревью архитектуры, как ArchUnit это решает |
| 2  | Подключение и базовая настройка | `02_setup/` | Зависимости, минимальный тест, конфигурация ClassFileImporter |
| 3  | Основы API | `03_core_api/` | JavaClass, ArchRule, предикаты, условия — строительные блоки правил |
| 4  | Правила для слоёв | `04_layer_rules/` | Layered architecture API, запрет обратных зависимостей |
| 5  | Правила именования | `05_naming_rules/` | Конвенции для классов, интерфейсов, пакетов, методов |
| 6  | Правила зависимостей | `06_dependency_rules/` | noClasses, forbidden imports, цикличные зависимости |
| 7  | Spring-специфичные правила | `07_spring_rules/` | Аннотации, бины, транзакции, контроллеры — что и как ограничивать |
| 8  | Кастомные правила | `08_custom_rules/` | DescribedPredicate, ArchCondition, сложные составные правила |
| 9  | Организация тестов | `09_test_organization/` | Базовые классы, переиспользование правил, разделение по модулям |
| 10 | CI и production практики | `10_ci_best_practices/` | Freeze violations, исключения, производительность, интеграция в пайплайн |

---

## Рекомендуемый порядок

Уроки 1–3 дают понимание инструмента и его API — без них остальные уроки сложно воспринимать. Уроки 4–8 можно читать в любом порядке после базы, уроки 9–10 — после практики с предыдущими.
